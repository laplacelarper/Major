# Side-Scan Sonar Renderer Fix

## Problem Statement

The synthetic sonar images generated by the original physics engine did not match the visual characteristics of real side-scan sonar images from the minehunting dataset. This mismatch caused poor domain alignment during transfer learning, reducing the effectiveness of Phase 2 fine-tuning on real data.

### Key Issues Identified

1. **Color Space Mismatch**
   - Synthetic: Grayscale (1 channel)
   - Real: RGB (3 channels)
   - Impact: Model trained on grayscale couldn't effectively transfer to RGB real data

2. **Intensity Calibration**
   - Synthetic: Too bright (mean ~0.5 normalized, ~127 in 0-255 range)
   - Real: Darker (mean ~40-80 in 0-255 range)
   - Impact: Different intensity distributions reduce transfer learning effectiveness

3. **Physics Model Oversimplification**
   - Synthetic: Simple Gaussian blobs for objects
   - Real: Complex acoustic signatures with multi-lobe patterns
   - Impact: Model learns incorrect object representations

4. **Noise Model Issues**
   - Synthetic: Uniform multiplicative noise
   - Real: Heteroscedastic noise (varies with intensity)
   - Impact: Model doesn't learn realistic noise characteristics

5. **Seabed Texture**
   - Synthetic: Generic procedural noise
   - Real: Specific seabed roughness and sediment patterns
   - Impact: Poor seabed representation in synthetic data

## Solution: New Side-Scan Sonar Renderer

Created `src/physics/sidescan_renderer.py` with a physics-accurate side-scan sonar image generator.

### Key Features

#### 1. Realistic Image Format
- **Resolution**: 512×512 (matches real data)
- **Color Space**: RGB (3 channels, matching real data)
- **Data Type**: uint8 (0-255 range, matching real data)
- **Format**: NumPy arrays (compatible with PyTorch)

#### 2. Accurate Physics Model
- **Range-Based Attenuation**: Exponential decay with distance (closer = brighter)
- **Frequency-Dependent Absorption**: Accounts for water absorption at sonar frequency
- **Realistic Object Signatures**: Gaussian blobs with acoustic shadows
- **Acoustic Shadows**: Cone-shaped shadows behind objects with proper falloff

#### 3. Realistic Noise Characteristics
- **Speckle Noise**: Rayleigh-distributed multiplicative noise (characteristic of coherent sonar)
- **Gaussian Noise**: Additive noise for sensor artifacts
- **Intensity-Dependent**: Noise adapts to local intensity

#### 4. Seabed Texture
- **Multi-Octave Noise**: Fractal-like texture using Perlin-like noise
- **Roughness Modulation**: Adjustable seabed roughness parameter
- **Natural Appearance**: Realistic seabed patterns

#### 5. Object Representation
- **Mine Signatures**: Bright spots (0.85 intensity) with strong shadows
- **Rock Signatures**: Moderately bright spots (0.65 intensity) with weaker shadows
- **Variable Sizes**: Objects can have different sizes (10-25 pixels)
- **Multiple Objects**: Support for multiple objects per image

### Implementation Details

#### SideScanParams Dataclass
```python
@dataclass
class SideScanParams:
    image_width: int = 512
    image_height: int = 512
    frequency_khz: float = 300.0  # Typical minehunting sonar
    base_intensity: float = 0.4   # Darker baseline
    attenuation_db_per_km: float = 15.0
    speckle_level: float = 0.3
    object_brightness_mine: float = 0.85
    object_brightness_rock: float = 0.65
    shadow_darkness: float = 0.15
```

#### SideScanRenderer Class
Main rendering pipeline:
1. Create seabed texture (multi-octave Perlin noise)
2. Apply range-based attenuation (exponential decay)
3. Add objects (mines/rocks with Gaussian signatures)
4. Add acoustic shadows (cone-shaped behind objects)
5. Add speckle noise (Rayleigh-distributed)
6. Add Gaussian noise (sensor artifacts)
7. Normalize to 0-255 range
8. Convert to RGB (grayscale → RGB by replicating channels)

#### Key Methods
- `render()`: Main rendering function
- `_create_seabed_texture()`: Multi-octave procedural noise
- `_generate_perlin_noise()`: Perlin-like noise generation
- `_apply_range_attenuation()`: Range-based intensity falloff
- `_add_object()`: Add mine/rock with shadow
- `_add_speckle_noise()`: Rayleigh-distributed noise
- `_add_gaussian_noise()`: Additive noise

### Integration with Physics Engine

Updated `src/physics/core.py` to support both renderers:

```python
class PhysicsEngine:
    def __init__(self, ..., use_realistic_renderer: bool = True):
        if use_realistic_renderer:
            self.sidescan_renderer = SideScanRenderer(params)
        else:
            self.renderer = SonarImageRenderer(image_size)  # Legacy
```

**Default**: `use_realistic_renderer=True` (new side-scan renderer)
**Fallback**: `use_realistic_renderer=False` (legacy renderer for compatibility)

### Usage Examples

#### Generate Single Image
```python
from src.physics.sidescan_renderer import SideScanRenderer

renderer = SideScanRenderer()
objects = [
    {'type': 'mine', 'x': 0.5, 'y': 0.5, 'size': 15},
    {'type': 'rock', 'x': 0.3, 'y': 0.7, 'size': 12}
]
image, label = renderer.render(objects=objects, random_seed=42)
# image: (512, 512, 3) uint8 array
# label: 1 (mine present)
```

#### Generate Dataset
```python
from src.physics.sidescan_renderer import generate_realistic_dataset

images, labels = generate_realistic_dataset(

    random_seed=42
)

# labels: (1000,) array with 0/1 values
```


### Impact on Training Pipeline

#### Phase 1: Synthetic Pretraining
- **Before**: Trained on grayscale, low-intensity synthetic images
- **After**: Trained on RGB, realistic-intensity synthetic images
- **Benefit**: Better feature learning from realistic synthetic data

#### Phase 2: Real Data Fine-Tuning
- **Before**: Poor domain alignment (grayscale → RGB, intensity mismatch)
- **After**: Better domain alignment (RGB → RGB, similar intensity)
- **Benefit**: More effective transfer learning from synthetic to real

#### Phase 3: Uncertainty Calibration
- **Before**: Calibrated on misaligned synthetic data
- **After**: Calibrated on well-aligned synthetic data
- **Benefit**: Better uncertainty estimates on real data

### Backward Compatibility

The new renderer is the default, but the legacy renderer is still available:

```python
# Use new side-scan renderer (default)
engine = PhysicsEngine(use_realistic_renderer=True)

# Use legacy renderer (for compatibility)
engine = PhysicsEngine(use_realistic_renderer=False)
```
